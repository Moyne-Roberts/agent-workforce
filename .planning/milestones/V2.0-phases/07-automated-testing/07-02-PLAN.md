**Phase 7: Execute experiments (3x per agent)**

Specify the experiment execution pipeline using `@orq-ai/evaluatorq` SDK:

1. **Per-agent experiment loop** (LOCKED: per-agent, not mega-experiment):
   - For each agent in the test set (or single filtered agent):
   - Create an evaluatorq job that invokes the deployed agent:
     ```
     job("invoke-{agent-key}", async (data) => {
       // Call deployed agent using @orq-ai/node SDK
       // Use agents.responses.create() (NOT deprecated agents.invoke())
       // Pass data.inputs.text as user message
       // Return agent response output
     })
     ```

2. **Triple-run execution** (LOCKED: 3 runs per agent):
   - For run in [1, 2, 3]:
     ```
     evaluatorq("test-{swarm}-{agent-key}-run-{N}", {
       data: { datasetId: testSplitDatasetId },
       jobs: [agentJob],
       evaluators: selectedEvaluators
     })
     ```
   - Use evaluatorq's built-in `parallelism` config to control concurrency
   - Add 2-second delay between runs to avoid rate limiting (research Pitfall 6)
   - Respect Retry-After headers on 429 responses

3. **Evaluator execution strategy:**
   - For LLM evaluators (coherence, helpfulness, relevance, instruction_following, harmfulness): use platform-side evaluation — let Orq.ai run the LLM scoring (research recommendation)
   - For function evaluators (json_validity, exactness, toxicity): use local evaluatorq scorers from `@orq-ai/evaluators` where available
   - If evaluatorq doesn't have a built-in scorer for a needed evaluator, fall back to platform-side evaluation

4. **Error handling per agent:**
   - If all 3 runs fail for an agent: record agent as "error" with failure reason, continue to next agent (LOCKED: don't block others)
   - If 1-2 runs fail but at least 1 succeeds: use available runs for scoring (note reduced confidence)
   - Capture experiment error details for inclusion in results

**Phase 8: Aggregate results and produce output**

1. **Triple-run aggregation** (LOCKED: median with variance):
   - For each agent, for each evaluator:
     ```
     scores = [run1_score, run2_score, run3_score]
     sorted = scores.sort()
     median = sorted[1]  // middle of 3
     mean = sum(scores) / 3
     variance = sum((s - mean)^2) / 3
     stddev = sqrt(variance)
     ci_95 = [max(0, median - 1.96 * stddev), min(scale_max, median + 1.96 * stddev)]
     ```
   - Note: CI scale_max depends on evaluator scale (1.0 for binary/continuous-01, 5.0 for continuous-15)

2. **Pass/fail determination** (LOCKED: per-evaluator thresholds):
   - Each evaluator has its own threshold (set in Phase 6 of pipeline)
   - Agent passes an evaluator if median score >= threshold
   - Agent overall pass = all evaluators pass
   - Overall test pass = all agents pass

3. **Category-sliced scoring** (LOCKED decision):
   - Group examples by category (happy-path, variation, boundary, adversarial, edge-case)
   - Compute per-evaluator scores within each category
   - This reveals patterns like "95% on happy-path but 40% on adversarial"

4. **Worst-performing cases** (LOCKED: bottom 3 per agent):
   - For each agent, find the 3 examples with lowest aggregate scores
   - Record: eval_id, input, expected_output, actual_output, per-evaluator scores, category, failure reason
   - Also count total failures (examples where any evaluator failed threshold)

5. **Output channel 1: test-results.json** (LOCKED: primary output for Phase 8):
   - Write results following the template schema in `orq-agent/templates/test-results.json`
   - Include all fields: test_run_id, swarm_name, timestamp, dataset metadata, evaluators, per-agent scores with medians/variance/CI, category_scores, worst_cases, total_failure_count, summary

6. **Output channel 2: test-results.md** (LOCKED: markdown for human review):
   - Generate markdown report with:
     - Header: swarm name, test date, dataset stats
     - Per-agent section: role, evaluators used, score table (evaluator | median | pass/fail | threshold), category breakdown table
     - Worst cases section: bottom 3 per agent in detail
     - Summary: overall pass/fail, total agents tested, agents passing, agents failing

7. **Output channel 3: Terminal summary table** (LOCKED decision):
   - Display concise table after test run:
     ```
     ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
      ORQ ► TEST RESULTS
     ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

     Agent              | Role    | Score  | Status
     -------------------|---------|--------|-------
     {agent-key}        | struct  | 0.85   | PASS
     {agent-key}        | conv    | 0.72   | FAIL

     Overall: 3/5 agents passing
     Details: test-results.md | JSON: test-results.json
     ```
   - Score shown is the lowest evaluator median for that agent (bottleneck score)
