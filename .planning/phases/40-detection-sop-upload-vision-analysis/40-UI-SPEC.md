## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, annotation label padding |
| sm | 8px | Compact element spacing, terminal entry internal gaps |
| md | 16px | Default element spacing, card content padding, terminal entry margin-bottom |
| lg | 24px | Section padding, annotation overlay panel gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical rhythm |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets for confirm/edit buttons in annotation overlay: minimum 44px height for accessibility on per-step actions
- Terminal panel dropzone: 96px minimum height (24px * 4) for comfortable drag target

## Typography

| Role | Size | Weight | Line Height | Usage in Phase 40 |
|------|------|--------|-------------|--------------------|
| Body | 14px | 400 (regular) | 1.5 | Terminal entry content, SOP preview prose, annotation step descriptions |
| Label | 12px | 500 (medium) | 1.4 | Timestamps, status labels, muted-foreground metadata, annotation bounding box labels |
| Heading | 18px | 600 (semibold) | 1.2 | Section titles ("Systems", terminal panel header, "Annotation Review") |
| Display | 24px | 600 (semibold) | 1.2 | Page titles (existing pattern: "Settings" at 24px/semibold) |

Notes:
- SOP markdown preview uses `prose prose-sm` (Tailwind typography plugin via react-markdown) which maps to 14px body with 1.625 line-height for readability

- All sizes match the existing codebase -- `text-sm` (14px), `text-xs` (12px), `text-lg` (18px), `text-2xl` (24px)

## Layout Contracts

### Terminal Panel (replaces Sheet drawer)

```
+------------------------------------------------------------------+
| Run Detail Page                                                    |
+-------------------------------------+----------------------------+
| SwarmGraph (primary view)           | Terminal Panel (right)      |
| React Flow graph                    | Width: 400px (same as old  |
| fills remaining viewport height     | Sheet drawer width)         |
|                                     |                            |
|                                     | [Entry Card] timestamp     |
|                                     |  status icon + message     |
|                                     |                            |
|                                     | [Entry Card] uploading     |
|                                     |  dropzone or paste area    |
|                                     |                            |
|                                     | [Entry Card] analyzing     |
|                                     |  skeleton + progress       |
|                                     |                            |
|                                     | [Open Overlay] button      |
|                                     |  for annotation review     |
+-------------------------------------+----------------------------+
```

- Terminal panel is always visible on the run detail page (not toggled like the old Sheet)
- Panel scrolls independently from the graph area
- Auto-scrolls to latest entry unless user has manually scrolled up
- "Jump to latest" floating button appears when user scrolls away from active entry
- On mount: fetches current state from DB (pipeline_steps + automation_tasks). Broadcast for real-time updates

### Annotation Review Overlay (full-width Dialog)

- Confirmed steps change highlight border to `border-green-500 bg-green-500/10`
- Number labels on highlights: `bg-blue-500 text-white text-xs px-1 rounded` positioned above the bounding box

## Interaction Contracts

### Terminal Panel Interactions

| Interaction | Behavior |
|-------------|----------|
| New entry appears | Card animates in with fade + slide-up (200ms ease-out). Panel auto-scrolls to bottom if user has not manually scrolled |
| User scrolls up | Auto-scroll pauses. "Jump to latest" button appears floating at bottom of panel |
| Click "Jump to latest" | Smooth scroll to latest entry, re-enable auto-scroll |
| Page mount / reconnection | Fetch current state from pipeline_steps + automation_tasks tables. Populate entries from DB, then subscribe to broadcast for live updates |
| Entry with rich UI | Dropzone, textarea, buttons render inline within the card. No separate modal for simple interactions |
| Complex interaction (annotation review) | "Review Steps" button in terminal entry opens the full-width Dialog overlay |

### SOP Upload Interactions

| Interaction | Behavior |
|-------------|----------|
| Tab between Upload/Paste | Tabs component toggles between file dropzone and textarea input |
| Drop .md file | File contents read client-side via FileReader, displayed as markdown preview |
| Paste markdown | Textarea input, "Submit SOP" button enabled when content is at least 10 characters |
| "Submit SOP" clicked | SOP text stored, new terminal entry shows rendered markdown preview |
| "Looks good" clicked | SOP confirmed. Next terminal entry prompts for screenshot upload |
| Edit after preview | User can go back and re-paste/re-upload before clicking "Looks good" |

### Screenshot Upload Interactions

| Interaction | Behavior |
|-------------|----------|
| Drop/select images | Thumbnails appear in a grid (3 columns, 80px thumbnails). Upload progress bar per file |
| Remove image | Click X on thumbnail, file removed from list, upload cancelled if in progress |
| Client-side resize | Images resized to max 1568px longest side via Canvas API before upload to Supabase Storage |
| "Submit Screenshots" | Files uploaded to Supabase Storage via signed URLs. Terminal shows "Analyzing..." entry with skeleton |
| Upload failure | Toast notification (sonner) with error. File marked with red border. User can retry individual files |

### Annotation Review Interactions

| Interaction | Behavior |
|-------------|----------|
| Overlay opens | Dialog fades in (shadcn default animation). Left panel shows SOP steps, right shows screenshots |
| Click SOP step | Corresponding screenshot scrolls into view on right panel. Matching annotations highlighted |
| Hover annotation highlight | Tooltip shows element details (type, label, confidence) |
| Click "Confirm" on step | Checkmark fills green. Highlight border changes from blue to green. Counter updates ("4/7 confirmed") |
| Click "Edit" on step | Inline text fields appear below step: action, target element, expected result. Save/Cancel buttons |
| Save edit | Fields collapse, step text updates, step remains unconfirmed (user must still click Confirm) |
| All steps confirmed | "Finalize" button enables (primary variant). Footer shows "7/7 steps confirmed" |
| Click "Finalize" | Loading state on button. AI re-analysis pass runs. Updated analysis replaces original. User reviews again if changes occurred |
| Re-analysis returns no changes | Toast: "Analysis confirmed -- no changes needed." Overlay closes. Pipeline proceeds |
| Re-analysis returns changes | Changed steps reset to unconfirmed. User must re-confirm changed steps. Toast: "AI updated 2 steps based on your corrections" |
| Close overlay (X or Escape) | Progress is preserved. User can reopen from the terminal panel entry |

### Systems Registry Interactions

| Interaction | Behavior |
|-------------|----------|
| Click "+ Add System" | Dialog opens with form: name (required), integration method (select), URL, auth notes, notes |
| Integration method select | Dropdown with 4 options: API, Browser Automation, Knowledge Base, Manual. Each with descriptive subtitle |
| Submit new system | Form validates (Zod), creates system via server action, toast confirmation, list refreshes |
| Click system row | Expand to show details (URL, auth notes, notes, linked projects count) |
| "Link to Project" | Dialog with project multi-select (same pattern as credential project linker) |
| "Delete System" | Confirmation dialog: "Delete [name]? This will unlink it from all projects." Destructive button |

### Missing Screenshot Warning

| Interaction | Behavior |
|-------------|----------|
| AI detects missing screens | Terminal entry card with amber border-left: "I noticed your SOP mentions a login screen but I don't see a matching screenshot. Would you like to add one?" |
| "Add Screenshot" button | Opens the screenshot upload entry again, user can add more |
| "Proceed Anyway" button | Warn toast: "Proceeding with incomplete screenshots -- automation may need more iterations." Pipeline continues |

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Focus management (annotation overlay) | shadcn Dialog handles focus trap, Escape to close, return focus to trigger |
| Keyboard navigation (terminal panel) | Tab through entry actions (buttons, inputs). Skip non-interactive entries |
| Screen reader (annotation highlights) | Each highlight has `aria-label` with step number and element description |
| Screen reader (terminal entries) | Each entry card has `role="log"` on the panel container, `aria-live="polite"` for new entries |
| Color contrast | All text meets WCAG AA 4.5:1 against respective backgrounds (oklch neutral palette) |
| Dropzone keyboard access | Dropzone area focusable with Enter/Space to trigger file picker |
| Status indicators | Never color-only -- all status badges include text labels alongside colored icons |
