## Interaction States

### PipelineSyncCard States

| State | Visual | Trigger |
|-------|--------|---------|
| Not checked | Gray dot, muted border, "Not checked" | Initial load, no sync data |
| Checking | Pulsing dot, muted border, "Checking..." | User clicks "Check Upstream" |
| Synced | Green dot, green border, "In sync" | All tracked files match manifest SHAs |
| Tier 1 drift | Amber dot, amber border, monitor description | Reference/template/command file changed |
| Tier 2 review | Amber dot, amber border, review description + link to GitHub issue | Agent structure changed |
| Tier 3 breaking | Red dot, red border, breaking description + link to GitHub issue | Agent added/removed/renamed |
| Error | Red dot, red border, error message | GitHub API call failed |

### Pipeline Blocked Banner

| Condition | Visual |
|-----------|--------|
| Any Tier 3 unresolved change | Full-width amber/red banner at top of run detail page, above the graph. Uses `bg-destructive/10 border border-destructive/20 text-destructive` pattern. Contains blocking message + link to /settings/health. |
| No blocking changes | Banner not rendered. |

### PipelineSyncSection Layout

```
+--------------------------------------------------+
| Pipeline Sync                    [Check Upstream] |
| Upstream pipeline agent compatibility             |
+--------------------------------------------------+
| [Sync Card: Agents]  [Sync Card: Refs]  [Sync..] |
| grid-cols-1 md:grid-cols-3 gap-4 mt-4            |
+--------------------------------------------------+
```

The section is rendered inside the existing Health tab on `/settings/health`, below the existing service health cards, separated by a `border-t` divider with `mt-8 pt-6` spacing.

### Sync Card Content Structure

Each card shows:
1. **Header row:** Status dot + card title (left), lucide icon (right) -- matches `HealthStatusCard`
2. **Status text:** "In sync" / tier description
3. **Timestamp:** "Last synced {relative time}" in `text-xs text-muted-foreground`
4. **Details (if drift/breaking):** Tier badge (`text-xs` inline rounded pill) + list of affected files in `text-xs text-muted-foreground`
5. **GitHub issue link (Tier 2-3 only):** "View issue" link in `text-xs text-primary`

### Tier Badge Styling

| Tier | Badge Color | Label |
|------|-------------|-------|
| Tier 0 | Not shown | -- |
| Tier 1 | `bg-muted text-muted-foreground` | "Monitor" |
| Tier 2 | `bg-amber-100 text-amber-800 dark:bg-amber-900/30 dark:text-amber-400` | "Review" |
| Tier 3 | `bg-red-100 text-red-800 dark:bg-red-900/30 dark:text-red-400` | "Breaking" |
